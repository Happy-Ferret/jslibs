jshost documentation

command line options

  • -w <0 or 1>
    Enable or disable the display of warnings.
  • -m  <maxbyte>
    Specifies the number of allocated bytes after which garbage collection is run.
  • -s <stacksize>
    Stack size for the script, the amount, in bytes, of private memory to allocate to the execution stack for the script.
  • -c <0 or 1>
    Enable or disable compiled script save, when enable, this will create a .jsxdr file for each executed .js file.
  • -u <0 or 1>
    [NOT IMPLEMENTED YET] Avoid any runtime checks, this is a kind of "release mode".
